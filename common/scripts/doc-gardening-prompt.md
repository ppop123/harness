# 文档维护 Prompt

> **使用方式**：对 Claude 说"更新文档"或"做文档整理"，Claude 会按照本文件的步骤执行。
> 建议频率：每周一次，与代码审计同时进行。

---

## 文档维护指令（复制给 Claude 执行）

```
请对这个项目的文档进行一次维护检查，确保文档与代码保持同步。

## 步骤 1：扫描新增内容

扫描最近的代码变更，检查是否有：
- 新增的业务实体（models/, types/ 目录）→ 应在 docs/domain-model.md 有对应记录
- 新增的 Service 函数 → 检查是否涉及新的业务流程，需要更新 domain-model.md
- 新增的技术依赖（package.json, pyproject.toml）→ 是否需要一条 ADR

## 步骤 2：检查文档陈旧性

检查 docs/ 目录中是否有：
- domain-model.md 中描述的实体，但代码中已不存在
- architecture.md 中的目录结构，但实际目录已变更
- tech-decisions/ 中的决策已经被推翻或替换

## 步骤 3：检查 AGENTS.md 是否需要更新

- 快速导航表格是否还准确？
- 禁止模式列表是否需要新增？
- 核心规则是否有变化？

## 步骤 4：生成文档更新报告

输出格式：

### 文档维护报告 - [日期]

**需要新增的内容**:
1. docs/domain-model.md 需要添加实体 [X]（因为代码里新增了 models/X.ts）
2. [...]

**需要更新的内容**:
1. docs/architecture.md 第 X 行的目录结构已过时
2. [...]

**可以删除的内容**:
1. domain-model.md 中的 [旧实体]，代码中已移除
2. [...]

## 步骤 5：执行更新

逐一询问我是否执行每个更新。对于明确的（比如删除已不存在的实体描述），可以直接更新。
```

---

## 文档质量标准

好的文档满足以下条件：

- **时效性**：描述的内容与当前代码一致
- **精准性**：术语与代码中的命名一致
- **可读性**：AI 读完后能理解业务概念，不需要读代码

> 记住 OpenAI 的原则：仓库是为 AI 的可读性优化的，人类是次要受众。
